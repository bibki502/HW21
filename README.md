### 1. Постановка задачи
Необходимо реализовать консольное приложение, моделирующее многопоточную обработку задач. Основные требования:
* Создание потокобезопасной очереди `TaskQueue`.
* Использование `std::mutex` и `std::condition_variable` для синхронизации.
* Запуск $N$ рабочих потоков для обработки 20 задач.
* Корректное завершение всех потоков без использования "busy-wait".


### 2. Описание реализации

#### Архитектура программы
Программа построена на взаимодействии главного потока (Producer) и пула рабочих потоков (Consumers).

* **Класс `TaskQueue`**: Является центральным узлом синхронизации. Внутри используется стандартный контейнер `std::queue<int>`. Доступ к данным ограничен мьютексом.
* **Потокобезопасность**: Для предотвращения состояния гонки (data race) все операции с очередью (`push`, `pop`, `front`) защищены объектами `std::lock_guard` или `std::unique_lock`. Это гарантирует, что только один поток может изменять структуру очереди в конкретный момент времени.
* **Механизм ожидания**: Вместо неэффективного цикла ожидания (busy-wait) использован `std::condition_variable`. Метод `cv.wait()` переводит поток в режим сна, если задач нет, и пробуждает его только при поступлении сигнала от главного потока (`notify_one`).
* **Завершение работы**: Реализовано через атомарный флаг `stop_flag`. Метод `stop()` устанавливает флаг в `true` и будит все потоки через `notify_all()`. Потоки, видя пустую очередь и поднятый флаг, выходят из цикла обработки и завершаются.
